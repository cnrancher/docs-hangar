Hangar 使用指引

快速上手​

参照以下指引，搭建一个 Demo Registry 镜像仓库，使用 Hangar 拷贝容器镜像。

  1. 搭建 Demo Registry 镜像仓库。

    docker run -d -p 5000:5000 --restart=always --name registry registry:2

  2. 在容器中运行 Hangar：

    docker run -it -v $(pwd):/hangar --network=host cnrancher/hangar:latest

  3. 创建镜像列表文件，用于将 Docker Hub 镜像拷贝（Mirror）至 Demo 镜像仓库：

    example_image_list.txt
    cnrancher/hangar:latest
    cnrancher/hangar:v1.9.0

    使用 Hangar Mirror 命令从 Docker Hub 拷贝镜像。

    hangar mirror \
        -f 'example_image_list.txt' \
        -s 'docker.io' \
        -d '127.0.0.1:5000' \
        --arch amd64,arm64 \
        --os linux \
        --tls-verify=false
    备注

    默认情况下，Demo Registry 镜像仓库可使用任意的用户名和密码登录。

    您可以参照 Distribution Registry Token Authentication 配置 Registry 的认证信息。

  4. 您可使用 Inspect 命令查看已拷贝的容器镜像 Manifest 索引：

    hangar inspect docker://127.0.0.1:5000/cnrancher/hangar:latest --raw --tls-verify=false
    {
      "mediaType": "application/vnd.docker.distribution.manifest.list.v2+json",
      "schemaVersion": 2,
      "manifests": [
        {
        "mediaType": "application/vnd.docker.distribution.manifest.v2+json",
    ......

配置文件​

Hangar 的配置文件与 containers/image 相同：

  • /etc/containers/policy.json: 默认的 Policy 配置文件。可使用 --insecure-policy 参数跳过策略检查步骤。

    默认的 Policy 配置文件可在 default-policy.json 获取。

命令​

  • mirror: 在镜像仓库之间拷贝容器镜像。
  • save: 将容器镜像从镜像仓库下载至压缩归档文件中。
  • load: 将容器镜像从压缩归档文件上传到镜像仓库中。
  • sync: 向压缩归档文件中增添容器镜像。
  • sign：容器镜像加签。
  • scan：容器镜像漏洞扫描。
  • view: 查看容器镜像安全信息。
  • archive：用于处理 Hangar 压缩包文件。
  • generate-list：用于生成 Rancher 镜像列表。 除此之外，Hangar 提供了一些高级指令，可访问 高级用法 页面获取更多信息。

支持的镜像仓库种类​

Hangar 兼容 API V2 容器镜像仓库，例如：

  • Registry
  • Docker Hub, quay.io, ghcr.io
  • Harbor 2.X
  • 公有云平台提供的镜像仓库服务，例如: AWS，Google Cloud，腾讯云 TCR、阿里云 ACR 等...

您需要确保容器镜像仓库支持以下格式的 Manifest 索引（mediaType）：

  • application/vnd.docker.distribution.manifest.list.v2+json
  • application/vnd.oci.image.index.v1+json
备注

Harbor 1.X 不支持上述 mediaType 格式，如需要使用 Hangar，请升级至 Harbor 2.X。
