Hangar Tutorial

Getting Started​

  1. Run hangar in cnrancher/hangar docker image:

    docker run -it -v $(pwd):/hangar cnrancher/hangar:latest

  2. Create an image list file for mirror images between registry servers:

    example_image_list.txt
    cnrancher/hangar:latest
    cnrancher/hangar:v1.7.0

    Use following command to mirror images from docker hub public registry to your destination registry server:

    hangar mirror \
        -f example_image_list.txt \
        -s docker.io \
        -d [DESTINATION_REGISTRY_URL] \
        --arch amd64,arm64 \
        --os linux

  3. You can use the inspect command to inspect the copied image manifest in destination registry:

    hangar inspect docker://[DESTINATION_REGISTRY_URL]/cnrancher/hangar:latest --raw
    {
      "mediaType": "application/vnd.docker.distribution.manifest.list.v2+json",
      "schemaVersion": 2,
      "manifests": [
        {
        "mediaType": "application/vnd.docker.distribution.manifest.v2+json",
    ...

Configuration​

The configuration file of hangar is compatible with containers/image configuration files.

  • /etc/containers/policy.json: The default trust policy file, use --insecure-policy option to skip policy check.

    The default policy file can be obtained at default-policy.json.

Commands​

  • mirror: Mirror container images between image registries.
  • save: Save container images from registry server into archive file.
  • load: Load container images from archive file to registry server.
  • sync: Sync (append) extra images into archive file.
  • validate: Subcommands of mirror/save/load/sync commands for validating the copied images.

Hanagr also provides some extra commands for advanced usage, see advanced-usage.

Supported Registries​

Hangar can work with API V2 container image registries:

  • Docker Registry
  • Harbor 2.X
  • Other Public Cloud Registries: Tencent Cloud TCR, Alibaba Cloud ACR, etc...

You need to ensure that the registry supports following mediaTypes:

  • application/vnd.docker.distribution.manifest.list.v2+json
  • application/vnd.oci.image.index.v1+json

Harbor 1.X does not support these mediaTypes, you need to upgrade to Harbor 2.X to use Hangar.
